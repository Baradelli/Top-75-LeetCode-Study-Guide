# Síntese: qual padrão usar?

> Quatro padrões de grafo cobertos. Esta seção é o ápice do raciocínio
> DFS/BFS que vem desde Matriz e Árvore — aqui ele aparece na estrutura mais
> geral. Hora de treinar o reflexo de escolher a ferramenta.

## A árvore de decisão da seção

```text
1. Preciso PERCORRER / alcançar / contar grupos conectados?
   ├── alcançar / cobrir / "existe caminho?"  → DFS ou BFS (com visitados)
   ├── caminho MAIS CURTO / menos passos       → BFS
   └── várias fontes ("tempo até alcançar tudo") → BFS MULTI-FONTE

2. Há DEPENDÊNCIAS / ordem ("X antes de Y") num grafo dirigido?
   → ORDENAÇÃO TOPOLÓGICA (Kahn) — também detecta ciclo

3. Preciso agrupar / "estão conectados?" / detectar ciclo (não-dirigido),
   especialmente com arestas chegando aos poucos?
   → UNION-FIND

4. É um GRID? → trate como grafo (vizinhos = arestas) e use os mesmos DFS/BFS
```

E os reflexos da seção: **sempre use visitados** (grafos têm ciclos);
**construa a lista de adjacência** a partir das arestas; e nomeie o Big-O como
**O(V + E)**.

## O mapa completo da seção

| Padrão | Pergunta-chave | Tempo |
| --- | --- | --- |
| DFS / BFS | "alcançar / contar componentes / clonar" | O(V + E) |
| BFS (mais curto) | "menor número de passos" | O(V + E) |
| Ordenação topológica | "ordem com dependências / tem ciclo?" | O(V + E) |
| Union-Find | "conectados? / ciclo / conexões dinâmicas" | ~O(V + E·α) |
| BFS multi-fonte | "tempo até alcançar tudo (grid)" | O(V + E) |

A ideia que une tudo: **um grafo é nós + arestas, e quase tudo é uma travessia
com visitados**. DFS (fundo), BFS (camadas), topológica (por dependências),
Union-Find (por grupos) — quatro formas de organizar essa travessia. Árvore,
grid e lista ligada foram ensaios para isto.
