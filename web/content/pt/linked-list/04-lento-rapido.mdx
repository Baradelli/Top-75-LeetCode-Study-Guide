> Dois ponteiros que andam pela lista em **velocidades diferentes** revelam

> **ciclo** — sem usar nenhuma memória extra.

> Tentou? Dá para contar o tamanho e andar metade — duas passadas. A teoria
> abaixo faz numa passada só, com uma ideia que rende muito mais que achar o
> meio.

## A teoria: velocidades diferentes

A ideia (também chamada "tartaruga e lebre"): dois ponteiros partem da cabeça,
mas `fast` anda **dois** nós por vez enquanto `slow` anda **um**.

```text
slow: +1 por passo
fast: +2 por passo
```

Como `fast` cobre o dobro do caminho, quando ele chega ao **fim**, `slow`
cobriu exatamente **metade** — ou seja, está no **meio**. Uma passada, sem
contar nada antes:

(Com número par de nós, este código devolve o **segundo** dos dois nós do

### Por que é O(n) tempo e O(1) espaço?

`fast` percorre a lista uma vez (em passos de 2), então o laço roda ~n/2 vezes,
cada uma O(1) → **O(n) de tempo**. Dois ponteiros, nenhuma estrutura extra →
**O(1) de espaço**. A alternativa "contar e andar metade" é O(n) também, mas
faz **duas** passadas.

## O grande truque: detecção de ciclo (Floyd)

E se a lista tiver um **ciclo** (algum `next` aponta para um nó anterior)? Um
laço `while curr` percorreria para sempre. Como detectar sem marcar nós nem
usar um conjunto (que custaria O(n) de memória)?

A sacada de **Floyd**: solte a tartaruga (`slow`, +1) e a lebre (`fast`, +2)
na pista. Se a pista é uma **reta** (sem ciclo), a lebre chega ao fim (`None`)
e acabou. Se a pista é **circular**, a lebre dá voltas e, cedo ou tarde,
**alcança a tartaruga por trás** — porque a cada passo a distância entre elas
diminui de 1. Se `fast == slow` em algum momento, há um ciclo.

<CodeRunner
  python={`class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def has_cycle(head):
    slow = head
    fast = head
    while fast and fast.next:
        slow = slow.next        # +1
        fast = fast.next.next   # +2
        if slow is fast:        # a lebre alcançou a tartaruga
            return True
    return False                # fast chegou ao fim: sem ciclo

# monta 3 -> 2 -> 0 -> -4 -> (volta para o nó de valor 2): tem ciclo
a = ListNode(3); b = ListNode(2); c = ListNode(0); d = ListNode(-4)
a.next = b; b.next = c; c.next = d; d.next = b   # ciclo!
print(has_cycle(a))    # True

# lista reta 1 -> 2 -> 3: sem ciclo
x = ListNode(1); y = ListNode(2); z = ListNode(3)
x.next = y; y.next = z
print(has_cycle(x))    # False`}
  javascript={`function ListNode(val, next) {
  this.val = val;
  this.next = next || null;
}
function hasCycle(head) {
  var slow = head;
  var fast = head;
  while (fast && fast.next) {
    slow = slow.next;        // +1
    fast = fast.next.next;   // +2
    if (slow === fast) {     // a lebre alcançou a tartaruga
      return true;
    }
  }
  return false;              // fast chegou ao fim: sem ciclo
}

// monta 3 -> 2 -> 0 -> -4 -> (volta ao nó de valor 2): tem ciclo
var a = new ListNode(3), b = new ListNode(2), c = new ListNode(0), d = new ListNode(-4);
a.next = b; b.next = c; c.next = d; d.next = b; // ciclo!
console.log(hasCycle(a)); // true

// lista reta 1 -> 2 -> 3: sem ciclo
var x = new ListNode(1), y = new ListNode(2), z = new ListNode(3);
x.next = y; y.next = z;
console.log(hasCycle(x)); // false`}
/>

### Por que Floyd funciona (e é O(1) de espaço)

Se há ciclo, uma vez que **as duas** estão dentro dele, a cada passo a lebre se
aproxima da tartaruga em 1 posição (anda 2, ela anda 1). A distância só
diminui, então em no máximo "tamanho do ciclo" passos elas se encontram. Se
**não** há ciclo, a lebre alcança o `None` e o laço termina. Custa **O(n) de
tempo** e **O(1) de espaço** — a alternativa óbvia (guardar nós já vistos num
set) também é O(n) de tempo, mas gasta O(n) de memória. Floyd não gasta nenhuma.

## Onde lento/rápido aparece

- **Meio da lista** (e daí: dividir a lista ao meio — base do merge sort em
  lista e do reorder).
- **Detecção de ciclo** e, num passo extra, achar **onde** o ciclo começa.
- **N-ésimo a partir do fim** (o gap fixo da aula anterior é um primo deste
  padrão).

## Resumo

- **Lento/rápido**: fast anda 2, slow anda 1 → slow para no **meio** quando
  fast acaba. Uma passada, O(n)/O(1).
- **Floyd (ciclo)**: se há ciclo, a lebre alcança a tartaruga; se não, a lebre
  acha o fim. O(n)/O(1), sem set.
- Base para dividir a lista, achar início do ciclo e "n-ésimo do fim".
