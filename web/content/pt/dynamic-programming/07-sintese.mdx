## A árvore de decisão de DP

```text
Reconheça a DP: há subproblemas que se REPETEM e a resposta se constrói dos menores?

1. Como modelar o estado?
   ├── uma posição numa sequência          → dp[i] (DP 1D)
   ├── um valor-alvo a compor com itens      → dp[a] (knapsack)
   ├── uma célula num grid                   → dp[r][c] (DP em grid)
   └── prefixos de DUAS sequências           → dp[i][j] (DP de strings)

2. Qual a recorrência (como dp se constrói dos estados menores)?
   ├── somar/contar caminhos                 → dp[i-1] + dp[i-2] / soma
   ├── decidir incluir ou excluir            → max(excluir, incluir)
   ├── minimizar com itens reutilizáveis     → min(dp[a-item] + 1)
   └── comparar últimos elementos            → casam? diagonal : vizinhos

3. Casos-base e ordem de preenchimento (dos menores para o maior).
```

E os reflexos da seção: **tempo ≈ estados × custo por estado**; quando a
recorrência olha só os últimos k estados, **reduza o espaço**; e, antes de
partir para DP, pergunte se um **guloso** já resolve.

## O mapa completo da seção

| Padrão | Estado | Tempo |
| --- | --- | --- |
| DP 1D | dp[i] (posição) | O(n) |
| Incluir/excluir | dp[i] = max(pular, pegar) | O(n) |
| Knapsack | dp[a] (valor-alvo) | O(alvo × itens) |
| DP em grid | dp[r][c] (célula) | O(m·n) |
| DP em sequências | dp[i][j] (prefixos) | O(m·n) |

A ideia-mãe: **DP é recursão + memória**. Defina o estado, a recorrência e os
casos-base, e o resto é preencher uma tabela. O que parecia o tema mais
assustador é, no fundo, o método mais sistemático do curso.
