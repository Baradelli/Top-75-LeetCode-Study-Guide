# Padrão 1: Contagem de caracteres (anagramas)

> Duas palavras são anagramas quando têm **exatamente os mesmos caracteres,
> em qualquer ordem**. A ferramenta que captura isso é o mapa de frequência —
> a "impressão digital" de uma string que ignora a ordem.

<Challenge id="valid-anagram" />

> Tentou? Comparar todas as permutações é O(k!) — inviável. Ordenar as duas
> strings e comparar funciona (O(k log k)), mas há algo ainda mais direto. A
> teoria abaixo mostra a "assinatura" que identifica anagramas em O(n).

## A teoria: a assinatura de frequência

Como saber se `"listen"` e `"silent"` são a mesma coisa rearranjada? Comparar
ordem não ajuda — a ordem é justamente o que muda. O que **não** muda é
*quantas vezes cada letra aparece*. Esse vetor de contagens é a assinatura:

```text
"listen" → {l:1, i:1, s:1, t:1, e:1, n:1}
"silent" → {l:1, i:1, s:1, t:1, e:1, n:1}   ← iguais ⇒ anagramas
```

É o mesmo hash map da seção Array, com uma pergunta diferente: em vez de "já
vi este valor?", perguntamos "quantas vezes vi cada caractere?". Duas strings
são anagramas se, e somente se, têm a mesma assinatura.

### Os sinais no enunciado

- A palavra **"anagrama"** aparece literalmente.
- "Reorganizar / permutar os caracteres de...".
- "Agrupar palavras que são rearranjos umas das outras".
- Qualquer coisa em que **a ordem não importa, só a composição**.

### Duas formas da assinatura

- **Mapa de contagens** (`{char: quantidade}`): geral, serve para qualquer
  alfabeto (Unicode inclusive).
- **String ordenada** (`sorted("listen") == "eilnst"`): mais simples de
  escrever, mas custa O(k log k) por palavra (k = tamanho). Útil como *chave*
  ao agrupar.

## Exemplo 1 — Valid Anagram

*"`s` e `t` são anagramas?"*

- **Força bruta:** testar todas as permutações de `s` — O(k!). Inviável.
- **O desperdício:** permutar é absurdo quando só precisamos saber se a
  *composição* é a mesma.
- **O padrão:** compare as assinaturas de frequência. Uma forma elegante numa
  passada só: **conte +1 para cada char de `s` e −1 para cada char de `t`**;
  no fim, todas as contagens têm que ser zero.

<CodeRunner
  python={`def is_anagram(s, t):
    if len(s) != len(t):
        return False
    contagem = {}
    for c in s:
        contagem[c] = contagem.get(c, 0) + 1
    for c in t:
        contagem[c] = contagem.get(c, 0) - 1
        if contagem[c] < 0:
            return False
    return True

print(is_anagram("listen", "silent"))  # True
print(is_anagram("rat", "car"))        # False
print(is_anagram("aacc", "ccac"))      # False`}
  javascript={`function isAnagram(s, t) {
  if (s.length !== t.length) {
    return false;
  }
  var contagem = {};
  for (var i = 0; i < s.length; i++) {
    var c = s[i];
    contagem[c] = (contagem[c] || 0) + 1;
  }
  for (var j = 0; j < t.length; j++) {
    var d = t[j];
    contagem[d] = (contagem[d] || 0) - 1;
    if (contagem[d] < 0) {
      return false;
    }
  }
  return true;
}

console.log(isAnagram("listen", "silent")); // true
console.log(isAnagram("rat", "car"));        // false
console.log(isAnagram("aacc", "ccac"));      // false`}
/>

### Por que é O(n)? (justificando)

- A verificação de tamanho corta na hora os casos triviais.
- Duas passadas (uma por string), cada uma O(n), com operações de mapa O(1)
  por caractere → **O(n) de tempo**.
- O mapa guarda no máximo o número de caracteres **distintos** do alfabeto —
  para letras minúsculas, no máximo 26. Espaço **O(1)** (constante, não
  cresce com n).

Compare com a abordagem "ordenar e comparar": `sorted(s) == sorted(t)`. Custa
O(n log n) — correta e curtíssima, mas mais lenta. Trade-off clássico:
simplicidade de código contra velocidade.

| Solução | Tempo | Espaço |
| --- | --- | --- |
| Testar permutações | O(k!) | — |
| Ordenar e comparar | O(n log n) | O(n) |
| Mapa de frequência | O(n) | O(1) (alfabeto fixo) |

## Exemplo 2 — Group Anagrams

*"Agrupe as palavras que são anagramas entre si."* Ex.: `["eat","tea","tan",
"ate","nat","bat"]` → `[["eat","tea","ate"],["tan","nat"],["bat"]]`.

- **O desperdício:** comparar cada palavra com todas as outras é O(n²)
  comparações de strings.
- **O padrão:** se anagramas têm a **mesma assinatura**, use a assinatura
  como **chave** de um hash map e jogue cada palavra no seu balde. Uma
  passada agrupa tudo.

A assinatura aqui costuma ser a *string ordenada* (`"eat"` → `"aet"`), porque
ela vira uma chave de dicionário naturalmente:

<CodeRunner
  python={`def group_anagrams(palavras):
    grupos = {}
    for palavra in palavras:
        chave = "".join(sorted(palavra))   # assinatura: letras ordenadas
        if chave not in grupos:
            grupos[chave] = []
        grupos[chave].append(palavra)
    return list(grupos.values())

print(group_anagrams(["eat", "tea", "tan", "ate", "nat", "bat"]))`}
  javascript={`function groupAnagrams(palavras) {
  var grupos = {};
  for (var i = 0; i < palavras.length; i++) {
    var chave = palavras[i]
    .split("")
    .sort(function (a, b) {
      return a < b ? -1 : a > b ? 1 : 0;
    })
    .join(""); // assinatura: letras ordenadas
    if (grupos[chave] === undefined) {
      grupos[chave] = [];
    }
    grupos[chave].push(palavras[i]);
  }
  var resultado = [];
  for (var k in grupos) {
    resultado.push(grupos[k]);
  }
  return resultado;
}

console.log(JSON.stringify(groupAnagrams(["eat","tea","tan","ate","nat","bat"])));`}
/>

### Por que é O(n · k log k)?

São `n` palavras; para cada uma, ordenar a assinatura custa O(k log k)
(k = tamanho da palavra). A inserção no hash map é O(k) (calcular o hash da
chave). Total: **O(n · k log k)**. Muito melhor que comparar todos os pares
(O(n² · k)). Se quiser cravar O(n · k), troque a assinatura ordenada por uma
contagem de 26 posições transformada em chave — mesma ideia, sem o `log`.

## Resumo

- A **assinatura de frequência** é a identidade de uma string sem a ordem.
- Valid Anagram: compare assinaturas (ou +1/−1 numa passada) — O(n), espaço
  O(1) no alfabeto fixo.
- Group Anagrams: use a assinatura como **chave** de hash map para agrupar —
  O(n·k log k), muito melhor que comparar pares.
