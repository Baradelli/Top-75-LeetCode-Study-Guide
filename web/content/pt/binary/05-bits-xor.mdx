# Padrão 4: O poder do XOR

> O XOR (`^`) tem uma propriedade quase mágica que resolve uma família inteira
> de problemas "achar o que sobra" em O(n) tempo e O(1) espaço — sem hash map,
> sem ordenar.

> Tentou? Provavelmente você usou um conjunto (set) ou a fórmula da soma —
> ambas funcionam! A teoria abaixo mostra uma terceira via, com XOR, que não
> usa memória extra nem corre risco de overflow.

## A teoria: as propriedades do XOR

O XOR compara bit a bit e dá 1 onde os bits **diferem**. Disso saem três
propriedades que, juntas, são uma ferramenta poderosa:

```text
1.  a ^ a = 0       (qualquer coisa com ela mesma se cancela)
2.  a ^ 0 = a       (XOR com zero não muda nada)
3.  a ^ b = b ^ a   (a ordem não importa — comutativo e associativo)
```

Junte tudo: se você fizer o XOR de uma porção de números, **tudo que aparece
em par se cancela** (propriedade 1) e sobra exatamente o que apareceu um
número ímpar de vezes. É um "detector de desemparelhados" que não guarda nada
na memória.

### Os sinais de XOR no enunciado

- "Todo elemento aparece **duas vezes**, exceto um" → o solitário sobra.
- "Um número está **faltando**" em uma sequência conhecida → emparelhe o que
  existe com o que deveria existir; o ausente sobra.
- "Troque dois valores sem variável auxiliar", "ache a diferença de bits".
- Restrição **O(1) de espaço** num problema que pareceria pedir um hash set.

## Exemplo 1 — Missing Number

*"Um array tem `n` números distintos de `0..n`; um está faltando."*

- **Força bruta / hash:** marque os presentes num set e veja qual falta —
  O(n) tempo mas O(n) espaço.
- **Soma:** `soma(0..n) − soma(nums)` dá o ausente — O(1) espaço, mas a soma
  pode estourar inteiros enormes.
- **XOR (sem overflow, sem memória):** faça o XOR de **todos os índices
  `0..n`** com **todos os valores**. Cada número presente aparece como índice
  *e* como valor → se cancela. O único índice sem valor correspondente é o que
  falta — e ele sobra.

### Por que é O(n) tempo e O(1) espaço?

Uma única passada pelos `n` elementos, com um XOR (O(1)) por passo →
**O(n) de tempo**. O estado é uma só variável acumuladora, não importa o
tamanho da entrada → **O(1) de espaço**. E, diferente da soma, o XOR nunca
"cresce" além da largura do inteiro — sem risco de overflow.

| Solução | Tempo | Espaço | Observação |
| --- | --- | --- | --- |
| Hash set | O(n) | O(n) | memória extra |
| Soma de Gauss | O(n) | O(1) | pode estourar inteiros |
| XOR | O(n) | O(1) | sem overflow, sem memória |

## Exemplo 2 — Single Number

*"Todo elemento aparece duas vezes, menos um. Ache o solitário."* É o XOR na
sua forma mais pura — sem nem precisar dos índices:

<CodeRunner
  python={`def single_number(nums):
    resultado = 0
    for x in nums:
        resultado = resultado ^ x   # pares se cancelam, sobra o solitário
    return resultado

print(single_number([4, 1, 2, 1, 2]))   # 4
print(single_number([7]))               # 7`}

## Resumo

- XOR: `a^a=0`, `a^0=a`, comutativo. Pares se cancelam; o desemparelhado sobra.
- **Single Number**: XOR de tudo a partir de 0 → o solitário. O(n)/O(1).
- **Missing Number**: XOR de índices `0..n` com os valores → o ausente sobra,
  sem overflow nem memória.
- Sinais: "aparece duas vezes menos um", "está faltando", O(1) de espaço.
