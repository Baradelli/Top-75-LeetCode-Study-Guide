# Síntese: qual padrão usar?

> Dois módulos, cinco padrões. Hora de treinar o reflexo: bater o olho num
> enunciado e saber se é busca binária, qual sabor, ou se é um problema de
> bits — com justificativa.

## A árvore de decisão da seção

```text
1. Os dados estão ORDENADOS (ou há uma fronteira monótona "não→sim")?
   ├── procurar um valor exato            → BUSCA BINÁRIA clássica
   ├── array ORDENADO mas GIRADO          → busca binária modificada
   │                                        (qual metade está ordenada?)
   ├── "primeiro/último que satisfaz..."  → busca binária por FRONTEIRA
   └── "a MENOR/MAIOR x que funciona"      → busca binária NA RESPOSTA
                                            (testar um x candidato é O(n))

2. O problema fala em BITS, binário, ou tem restrição estranha de espaço?
   ├── "aparece 2x menos um" / "faltando" → XOR (pares se cancelam)
   ├── contar/olhar/ligar/apagar bits     → operadores & | ^ ~ << >>
   ├── "para cada i, conte os bits"       → counting bits (ans[i]=ans[i>>1]+(i&1))
   └── "sem usar +", inverter bits        → técnicas (carry por &<<1, espelho)
```

E o teste de sanidade de sempre: se o enunciado pede **melhor que O(n)** num
dado ordenado, é quase certo que a resposta é **O(log n)** — busca binária.

## O mapa completo da seção

| Padrão | Pergunta-chave | Tempo | Espaço |
| --- | --- | --- | --- |
| Busca binária clássica | "achar valor em array ordenado" | O(log n) | O(1) |
| Busca binária girada | "array ordenado e girado" | O(log n) | O(1) |
| Busca na resposta | "menor/maior x que satisfaz" | O(n log(faixa)) | O(1) |
| XOR | "aparece 2x menos um / faltando" | O(n) | O(1) |
| Operadores de bit | testar/ligar/apagar/contar bits | O(1)–O(largura) | O(1) |
| Counting bits | "bits 1 de 0..n" | O(n) | O(n) (saída) |

A ideia que une o módulo de busca binária: **descartar metade com certeza a
cada passo**. A que une o módulo de bits: **operar nos 0s e 1s direto**, em vez
de tratar o número como quantidade.
