# Síntese: qual padrão usar?

> Você conhece os 5 padrões. Falta a habilidade que vale a entrevista:

## A árvore de decisão da seção

Diante de um problema de array, percorra estas perguntas em ordem:

```text
1. A pergunta é sobre um TRECHO CONTÍGUO (subarray)?
   ├── Sim, e os valores são todos positivos (ou janela de tamanho fixo)
   │     → SLIDING WINDOW (estado entra/sai pelas bordas)
   ├── Sim, mas há NEGATIVOS
   │     ├── "maior soma/produto"      → KADANE (estender ou recomeçar)
   │     └── "quantos somam k" /
   │         perguntas de intervalo    → PREFIX (+ hash map para contar)
   └── Não, continue ↓

2. Procuro PARES/TRIOS de elementos?
   ├── Preciso de valores EXATOS (complemento) e/ou dos índices originais
   │     → HASH MAP (pergunte ao passado, registre para o futuro)
   └── Array ordenado (ou posso ordenar) / condição de DESIGUALDADE
         → TWO POINTERS (descarte provado em bloco)

3. Pergunto repetidamente "já vi? quantas vezes? existe?"
      → HASH MAP

4. "Para cada posição, algo sobre TODO o resto" (esquerda × direita)
      → PREFIX + SUFFIX (duas passadas)
```

E o teste de sanidade final, sempre: **os limites**. n até 10⁵-10⁶ pede
O(n) ou O(n log n) — se o seu plano é O(n²), volte à árvore.

## O mapa completo da seção

| Padrão | Desperdício que elimina | Pergunta-chave | Tempo | Espaço |
| --- | --- | --- | --- | --- |
| Hash map | busca repetida | "já vi o que procuro?" | O(n) | O(n) |
| Two pointers | pares sem chance | "qual lado abandono — e por quê?" | O(n)* | O(1) |
| Sliding window | recálculo de janelas | "o que entra, o que sai?" | O(n) | O(1)** |
| Prefix/Suffix | resoma de trechos | "posso pré-acumular?" | O(n) | O(1)-O(n) |
| Kadane | reavaliar todo começo | "estendo ou recomeço?" | O(n) | O(1) |

\* + O(n log n) se precisar ordenar antes. &nbsp;&nbsp; \*\* depende do
estado da janela (um escalar = O(1); um mapa de contagens = O(k)).

Repare no que todos têm em comum: **uma passada (ou duas) com trabalho O(1)
por elemento**, cada um com seu truque para nunca revisitar trabalho feito.
Esse é o espírito de tudo que derrota O(n²).
