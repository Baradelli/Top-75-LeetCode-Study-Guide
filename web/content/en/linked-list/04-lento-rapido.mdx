# Pattern 3: Slow and fast pointers

> Two pointers walking through the list at **different speeds** reveal
> surprising things: the middle of the list in a single pass, and whether a
> **cycle** exists — without using any extra memory.

<Challenge id="middle-of-list" />

> Did you try? You can count the size and walk halfway — two passes. The theory
> below does it in a single pass, with an idea that pays off far more than just
> finding the middle.

## The theory: different speeds

The idea (also called "tortoise and hare"): two pointers start at the head, but
`fast` moves **two** nodes at a time while `slow` moves **one**.

```text
slow: +1 per step
fast: +2 per step
```

Since `fast` covers twice the distance, when it reaches the **end**, `slow` has
covered exactly **half** — that is, it is at the **middle**. One pass, without
counting anything first:

### Why is it O(n) time and O(1) space?

`fast` traverses the list once (in steps of 2), so the loop runs ~n/2 times,
each one O(1) → **O(n) time**. Two pointers, no extra structure → **O(1)
space**. The "count and walk halfway" alternative is O(n) too, but makes
**two** passes.

## The big trick: cycle detection (Floyd)

What if the list has a **cycle** (some `next` points back to an earlier node)? A
`while curr` loop would run forever. How do you detect it without marking nodes
or using a set (which would cost O(n) memory)?

**Floyd's** insight: release the tortoise (`slow`, +1) and the hare (`fast`, +2)
onto the track. If the track is **straight** (no cycle), the hare reaches the
end (`None`) and that's that. If the track is **circular**, the hare keeps going
around and, sooner or later, **catches up to the tortoise from behind** —
because at each step the distance between them shrinks by 1. If `fast == slow`
at some point, there is a cycle.

<CodeRunner
  python={`class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def has_cycle(head):
    slow = head
    fast = head
    while fast and fast.next:
        slow = slow.next        # +1
        fast = fast.next.next   # +2
        if slow is fast:        # the hare caught the tortoise
            return True
    return False                # fast reached the end: no cycle

# builds 3 -> 2 -> 0 -> -4 -> (back to the node with value 2): has a cycle
a = ListNode(3); b = ListNode(2); c = ListNode(0); d = ListNode(-4)
a.next = b; b.next = c; c.next = d; d.next = b   # cycle!
print(has_cycle(a))    # True

# straight list 1 -> 2 -> 3: no cycle
x = ListNode(1); y = ListNode(2); z = ListNode(3)
x.next = y; y.next = z
print(has_cycle(x))    # False`}

  this.val = val;
  this.next = next || null;
}
function hasCycle(head) {
  var slow = head;
  var fast = head;
  while (fast && fast.next) {
    slow = slow.next;        // +1
    fast = fast.next.next;   // +2
    if (slow === fast) {     // the hare caught the tortoise
      return true;
    }
  }
  return false;              // fast reached the end: no cycle
}

// builds 3 -> 2 -> 0 -> -4 -> (back to the node with value 2): has a cycle
var a = new ListNode(3), b = new ListNode(2), c = new ListNode(0), d = new ListNode(-4);
a.next = b; b.next = c; c.next = d; d.next = b; // cycle!
console.log(hasCycle(a)); // true

// straight list 1 -> 2 -> 3: no cycle
var x = new ListNode(1), y = new ListNode(2), z = new ListNode(3);
x.next = y; y.next = z;
console.log(hasCycle(x)); // false`}

### Why Floyd works (and is O(1) space)

If there is a cycle, once **both** are inside it, the hare gets 1 position
closer to the tortoise at every step (it moves 2, the tortoise moves 1). The
distance only shrinks, so within at most "cycle length" steps they meet. If
there is **no** cycle, the hare reaches `None` and the loop ends. It costs
**O(n) time** and **O(1) space** — the obvious alternative (storing seen nodes
in a set) is also O(n) time, but spends O(n) memory. Floyd spends none.

## Where slow/fast shows up

- **Middle of the list** (and from there: splitting the list in half — the basis
  of merge sort on lists and of reorder).
- **Cycle detection** and, with one extra step, finding **where** the cycle
  begins.
- **N-th from the end** (the fixed gap from the previous lesson is a cousin of
  this pattern).

## Summary

- **Slow/fast**: fast moves 2, slow moves 1 → slow stops at the **middle** when
  fast runs out. One pass, O(n)/O(1).
- **Floyd (cycle)**: if there is a cycle, the hare catches the tortoise; if not,
  the hare finds the end. O(n)/O(1), no set.
- The basis for splitting the list, finding the start of the cycle, and "n-th
  from the end".
