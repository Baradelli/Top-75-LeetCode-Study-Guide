# Pattern 1: Dummy node and two pointers

> Two tools that show up in almost every list problem: the **sentinel node**
> (dummy), which removes the special case "what if it's the head?", and the
> **two pointers with a fixed gap**, which solve "from the end" in one pass.

<Challenge id="remove-nth-from-end" />

> Did you try? The natural approach is to count the list size in one pass and,
> in another, walk to the right node — **two passes**, and still handle removing
> the head as a separate case. The theory below does it all in one pass and with
> no special case.

## Theory 1: the dummy node (sentinel)

Operations on the **head** are annoying because the head has no "previous node"
to relink. The result: a bunch of `if head is None` and `if previous is None`
scattered around.

The **dummy** trick: create a fake node that points to the real head.

```text
dummy ──▶ [1] ──▶ [2] ──▶ [3] ──▶ ∅
```

Now **every** real node has a previous one (in the worst case, the dummy). You
operate without special cases and, at the end, return `dummy.next` — which is
the head (possibly a new one). Add one paper node, get branch-free code.

## Theory 2: two pointers with a fixed gap

For questions like *"the n-th from the end"*, you don't know where the end is
without traversing. The insight: keep two pointers with a **fixed gap of n
nodes** between them.

1. Advance `fast` **n steps** ahead.
2. Move `fast` and `slow` **together** until `fast` reaches the end.
3. Since the distance between them is n, when `fast` runs out, `slow` is
   exactly **n nodes before the end** — right where you need it to relink.

One pass, without counting the size first. Combined with the dummy (for the case
of removing the head), the code stays clean:

    def __init__(self, val=0, next=None):

def remove_nth_from_end(head, n):

    slow = dummy

        fast = fast.next

        fast = fast.next
        slow = slow.next
    slow.next = slow.next.next  # slow is right before the target
    return dummy.next

        c = c.next

def to_list(head):
    r = []

        head = head.next

function removeNthFromEnd(head, n) {
  var dummy = new ListNode(0, head); // sentinel before the head
  var fast = dummy;
  var slow = dummy;
  for (var i = 0; i < n; i++) {      // open a gap of n nodes
    fast = fast.next;
  }
  while (fast.next) {                // move both to the end
    fast = fast.next;
    slow = slow.next;
  }
  slow.next = slow.next.next;        // slow is right before the target
  return dummy.next;
}

function build(arr) {
  var d = new ListNode(0), c = d;
  for (var i = 0; i < arr.length; i++) { c.next = new ListNode(arr[i]); c = c.next; }
  return d.next;
}
function toList(head) {
  var r = [];
  while (head) { r.push(head.val); head = head.next; }
  return r;
}

console.log(toList(removeNthFromEnd(build([1]), 1)));          // []  (removed the head)`}

### Why is it O(n) time and O(1) space?

Each pointer traverses the list **once** (`fast` walks n + the rest; `slow`
walks the rest) — O(n) total, a single pass. We use only a few pointers, no
auxiliary array → **O(1) space**. The "count and then walk" version is also
O(n), but makes **two** passes and needs the special case for the head (which
the dummy eliminates).

## When to use each one

- **Dummy node:** whenever the operation might touch the **head** (remove,
  insert at the start, merge). It costs one node and removes a bunch of `if`s.
- **Two pointers with a gap:** "n-th from the end", "remove the middle one", any
  "position relative to the end" in one pass.

## Summary

- **Dummy node**: a sentinel node before the head removes the special cases of
  operating at the start; return `dummy.next`.
- **Two pointers with a gap of n**: solves "from the end" in one pass, without
  counting the size.
- Together: O(n) time, O(1) space, no ugly branches.
