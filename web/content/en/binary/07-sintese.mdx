# Synthesis: which pattern to use?

> Two modules, five patterns. Time to train the reflex: glance at a statement
> and know whether it's binary search, which flavor, or whether it's a bit
> problem — with justification.

## The section's decision tree

```text
1. Is the data SORTED (or is there a monotonic "no→yes" boundary)?
   ├── search for an exact value             → classic BINARY SEARCH
   ├── SORTED but ROTATED array              → modified binary search
   │                                           (which half is sorted?)
   ├── "first/last that satisfies..."        → BOUNDARY binary search
   └── "the SMALLEST/LARGEST x that works"   → binary search ON THE ANSWER
                                               (testing a candidate x is O(n))

2. Does the problem mention BITS, binary, or have a strange space constraint?
   ├── "appears 2x except one" / "missing"   → XOR (pairs cancel out)
   ├── count/look at/turn on/clear bits      → operators & | ^ ~ << >>
   ├── "for each i, count the bits"          → counting bits (ans[i]=ans[i>>1]+(i&1))
   └── "without using +", reverse bits       → techniques (carry via &<<1, mirror)
```

And the usual sanity check: if the statement asks for **better than O(n)** on
sorted data, it's almost certain the answer is **O(log n)** — binary search.

## The section's complete map

| Pattern | Key question | Time | Space |
| --- | --- | --- | --- |
| Classic binary search | "find a value in a sorted array" | O(log n) | O(1) |
| Rotated binary search | "sorted and rotated array" | O(log n) | O(1) |
| Search on the answer | "smallest/largest x that satisfies" | O(n log(range)) | O(1) |
| XOR | "appears 2x except one / missing" | O(n) | O(1) |
| Bit operators | test/turn on/clear/count bits | O(1)–O(width) | O(1) |
| Counting bits | "1 bits of 0..n" | O(n) | O(n) (output) |

The idea that unites the binary search module: **discard half with certainty
at each step**. The one that unites the bits module: **operate on the 0s and
1s directly**, instead of treating the number as a quantity.
