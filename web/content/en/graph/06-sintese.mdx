# Synthesis: which pattern to use?

> Four graph patterns covered. This section is the peak of the DFS/BFS
> reasoning that has been coming since Matrix and Tree — here it appears on the
> most general structure. Time to train the reflex of choosing the tool.

## The section's decision tree

```text
1. Do I need to TRAVERSE / reach / count connected groups?
   ├── reach / cover / "is there a path?"        → DFS or BFS (with visited)
   ├── SHORTEST path / fewest steps              → BFS
   └── several sources ("time to reach everything") → MULTI-SOURCE BFS

2. Are there DEPENDENCIES / order ("X before Y") in a directed graph?
   → TOPOLOGICAL SORT (Kahn) — also detects a cycle

3. Do I need to group / "are they connected?" / detect a cycle (undirected),
   especially with edges arriving gradually?
   → UNION-FIND

4. Is it a GRID? → treat it as a graph (neighbors = edges) and use the same DFS/BFS
```

And the section's reflexes: **always use visited** (graphs have cycles);
**build the adjacency list** from the edges; and name the Big-O as **O(V + E)**.

## The section's complete map

| Pattern | Key question | Time |
| --- | --- | --- |
| DFS / BFS | "reach / count components / clone" | O(V + E) |
| BFS (shortest) | "fewest number of steps" | O(V + E) |
| Topological sort | "order with dependencies / is there a cycle?" | O(V + E) |
| Union-Find | "connected? / cycle / dynamic connections" | ~O(V + E·α) |
| Multi-source BFS | "time to reach everything (grid)" | O(V + E) |

The idea that ties it all together: **a graph is nodes + edges, and almost
everything is a traversal with visited**. DFS (depth), BFS (layers),
topological (by dependencies), Union-Find (by groups) — four ways to organize
that traversal. Tree, grid and linked list were rehearsals for this.
