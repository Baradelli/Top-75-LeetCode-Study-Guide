# Synthesis: which pattern to use?

> Five tree patterns covered. Time to train the reflex: read a tree problem and
> know which tool to reach for — with justification.

## The section's decision tree

```text
1. Does the answer COMBINE from the subtrees (height, sum, validate, invert)?
   → RECURSIVE DFS (base case None → solve left → solve right → combine)

2. Is the question per LEVEL (values of each level, side view, minimum depth)?
   → Level-order BFS (queue + level-size trick)

3. Is it a BST (left < node < right)?
   ├── search / LCA / insert   → descend only one side: O(h)
   └── "k-th smallest" / sorted → IN-ORDER traversal (ascending)

4. Do I need to REBUILD or SERIALIZE the tree?
   → preorder gives the root; empty markers make the string reconstructible

5. Is the problem about PREFIXES or a dictionary of words?
   → TRIE (per-character path; O(L) per operation)
```

And the reflex that holds for the whole section: **think recursively**. Define
the base case, trust that the recursion solves the children, and focus on the
combination. Most trees yield to that habit.

## The complete map of the section

| Pattern | Key question | Time | Space |
| --- | --- | --- | --- |
| Recursive DFS | "combine the children's result" | O(n) | O(h) |
| Level-order BFS | "per level / minimum depth" | O(n) | O(width) |
| BST (search/LCA) | "left &lt; node &lt; right" | O(h) | O(h) |
| BST in-order | "k-th / sorted" | O(n) | O(h) |
| Construction/serialization | "rebuild without ambiguity" | O(n) | O(h) |
| Trie | "prefixes / dictionary" | O(L) per op. | O(total chars) |

Notice the recurring **O(h)**: the height governs the recursion's space and the
time of the BST search. Balanced → log n; degenerate → n. This is the section's
deep dive into Big-O — whenever you say O(h), know what h means.
