# Synthesis: which pattern to use?

> Five matrix patterns covered. Time to train the reflex: read a grid problem
> and know which tool to pull out — with justification.

## The section's decision tree

```text
1. Do I need to TRAVERSE/PRODUCE the matrix in a special order (spiral, layers)?
   → LAYER-BY-LAYER TRAVERSAL (four shrinking bounds)

2. Do I need to TRANSFORM the matrix without extra memory (rotate, zero)?
   → IN-PLACE: decompose (transpose + reverse) and/or use the 1st row/column
     as markers

3. Is the question about CONNECTED REGIONS (islands, areas, filling)?
   → DFS / FLOOD FILL (permanent mark; covers the region)

4. Do I need to find/count specific PATHS (a word, all paths)?
   → BACKTRACKING (DFS with a temporary mark: mark → explore → unmark)

5. Is the question "the SHORTEST path / fewest steps"?
   → BFS (queue, layers of increasing distance)
```

And the reflexes for the whole section: **name both dimensions in the Big-O**
(O(m·n), not O(n²)), **always check the bounds** before accessing a neighbor,
and remember that **a grid is a graph** — DFS/BFS are the core tools.

## The complete map of the section

| Pattern | Key question | Structure | Time | Space |
| --- | --- | --- | --- | --- |
| Spiral traversal | "traverse in a spiral/layers" | 4 bounds | O(m·n) | O(1) |
| In-place transformation | "rotate/zero without a copy" | indices + markers | O(m·n) | O(1) |
| DFS / flood fill | "connected regions" | stack (recursion) | O(m·n) | O(m·n) |
| Backtracking | "specific path / all" | DFS + undo | O(m·n·4ᴸ) | O(L) |
| BFS | "shortest path" | queue | O(m·n) | O(m·n) |

The idea that ties half the section together: **the grid is a graph**, and DFS
(deep), backtracking (deep + undo) and BFS (wide) are three ways to traverse
it, each for a kind of question. You leave here ready for the **Graphs**
section, where these same algorithms appear without the "crutch" of the regular
grid.

hard — with hidden cases. The hard one (capturing surrounded regions) calls for
the insight of **inverting the problem** and running DFS from the edges:
exactly the kind of reasoning the decision tree above trains.
